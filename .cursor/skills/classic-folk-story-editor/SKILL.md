# Classic Folk Story Editor

中国经典民间故事专用评测技能。针对 classic-folk-story 技能产出的故事重述，提供量化评分和可执行的修改方案。

**路径说明：** 本技能内所有 `references/` 路径均相对于本 skill 根目录（即 `classic-folk-story-editor/`）。

**设计特点：** 专为经典民间故事重述优化——7 个专项维度（含忠实度、氛围营造），权重按故事分类调整，内置风格卡合规检查。

## 评测工作流

```
1. 接收材料 → 2. 通读分析 → 3. 逐维度评分 → 4. 风格卡合规检查 → 5. 生成评测报告 → 6. 输出修改清单
```

### 步骤 1：接收材料

由 classic-folk-story 工作流通过 subagent 调用时传入，包含：

- **正文路径**（必需）：故事文件路径
- **故事分类**（必需）：上古神话 / 四大民间传说 / 仙凡奇缘 / 聊斋志异 / 志怪小说 / 搜神志怪 / 子不语·阅微 / 民间恐怖与悬疑
- **篇幅档位**（必需）：短篇 2000–3500 / 中篇 4000–6000 / 长篇 7000–10000
- **使用的风格**（必需）：汪曾祺（默认）/ 汪曾祺·暗面 / 白话演绎 / 莫言式 / 东野圭吾式
- **作品阶段**：初稿（默认）/ 修订稿
- **大纲 / 设定路径**（可选）

无需向用户确认参数——所有参数由上游工作流提供，缺失项取默认值。

### 步骤 2：通读分析

完整阅读正文，建立整体印象：
- 概括故事梗概（2-3 句）
- 识别所用的叙述结构（四段式 / 两段式 / 三段式 / 反转式）
- 标记初步发现的亮点和问题点
- 判断明面 / 暗面的调性是否与故事分类匹配

### 步骤 3：逐维度评分

从七个维度逐一评测打分（1-10 分）。每个维度的详细评分标准参见 [references/rubric.md](references/rubric.md)。

**七大评测维度：**

| # | 维度 | 核心考察点 |
|---|------|-----------|
| 1 | 忠实度 | 人物 / 情节 / 出处与参考文件一致性，关键节点无遗漏无篡改 |
| 2 | 结构与节奏 | 所选结构是否合理，铺垫 / 高潮 / 收束的比例，快慢交替 |
| 3 | 风格执行 | 风格卡合规度，修辞节制，语言质地，叙述声音一致性 |
| 4 | 氛围营造 | 闲笔效果，感官细节，环境描写，明面 / 暗面的调性准确度 |
| 5 | 角色与对话 | 人物是否通过细节立住，对话是否自然有性格，配角质量 |
| 6 | 情感与余韵 | 情感是否自然传达（非强行煽情），结尾画面是否留有余韵 |
| 7 | 可读性 | 开篇吸引力，阅读流畅度，翻页动力，信息密度 |

**评分规则：**
- 每个维度给出整数或 0.5 精度的分数
- 必须给出分数理由和具体文本引用
- 指出该维度的最大亮点和最大问题
- 根据故事分类应用对应权重计算综合分（权重见 [references/category-focus.md](references/category-focus.md)）

### 步骤 4：风格卡合规检查

根据上游指定的风格（默认为汪曾祺），执行对应的自检清单。

**汪曾祺·明面——检查项：**

| 检查项 | 判定 |
|--------|------|
| 句子是否干净（能删的字删了）| 合格 / 偏冗 |
| 有无排比 | 无 / 有（需删） |
| "像…"明喻频次 | 正常 / 偏高 |
| 形容词堆砌 | 无 / 有 |
| 至少一处食物或日常细节 | 有 / 缺 |
| 感官偏淡偏暖 | 是 / 偏浓 |
| 闲笔（看似无关的世界细节）| 有 / 缺 |
| 对话简短、自然、有性格 | 是 / 偏长或趋同 |
| 结尾为画面或细节收束 | 是 / 有说教 / 有总结 |
| 有无抒情段落 | 无 / 有（需删） |
| 有无大段内心独白 | 无 / 有 |

**汪曾祺·暗面——额外检查项：**

| 检查项 | 判定 |
|--------|------|
| 温度是否"凉"而非"冷酷" | 是 / 过冷 |
| 闲笔是否为冷场景（空巷、旧物、夜路）| 是 / 仍为暖场景 |
| 恐怖感靠具体感官还是形容词堆砌 | 感官 / 堆砌 |
| 至少一处"静得不对劲"的描写 | 有 / 缺 |
| 结尾画面"静的、空的、有一点不对" | 是 / 偏暖 |
| 对话够少够短，鬼的台词"正常得不对劲"或"只说几个字" | 是 / 偏多 |

其他风格（白话演绎 / 莫言 / 东野圭吾）的合规检查，按对应风格卡中的自检清单执行。

### 步骤 5：生成评测报告

**篇幅**：若评测对象为文件，须先调用 MCP 工具 **count_document(正文路径)** 获取精确字数，勿估算。

按以下模板输出：

```

## 基本信息
- 故事名：
- 分类：[八类之一]
- 篇幅：X 字（档位：短篇 / 中篇 / 长篇）
- 风格：汪曾祺·明面 / 汪曾祺·暗面 / 白话演绎 / 莫言式 / 东野圭吾式
- 结构：四段式 / 两段式 / 三段式 / 反转式
- 作品阶段：初稿 / 修订稿
```

### 步骤 6：输出修改清单

修改建议按优先级分三级：

```
## 修改清单

### 🔴 必改（影响故事根基）
- **问题**：具体描述
- **位置**：定位到段落
- **改法**：可执行的修改方案
- **示例**：修改示范（保持作者所用风格）

### 🟡 建议（显著提升）
格式同上

### 🟢 打磨（锦上添花）
格式同上
```

**修改清单原则：**
- 每条必须具体、可执行，禁止空泛的"加强描写""深化主题"
- 指出问题时必须引用原文
- 修改示范必须保持上游选定的风格，不替换为评测者自己的风格
- 忠实度问题（篡改情节、弄错人物）一律归入 🔴 必改

**输出**：以 Markdown 写入 `novel/作品名-评测报告.md`（与正文同目录），并将综合评分、三大亮点、三大问题、修改清单摘要返回给主对话。

## 评分校准原则

- **锚定参照系**：以汪曾祺原作为 9-10 分锚点，当代优秀改编为 7-8 分
- **避免集中效应**：不要所有维度给 6-7 分，大胆拉开差距
- **阶段适配**：初稿评测聚焦忠实度、结构和风格执行，不苛求文笔细节；修订稿全面严格
- **风格公平**：不因个人偏好贬低非默认风格（莫言式 / 东野式各有其标准）
- **正向优先**：先认可亮点再指出问题
