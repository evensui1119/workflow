# Novel Editor

小说多维评测与编辑建议技能，提供量化评分和可执行的修改方案。

**路径说明：** 本技能内所有 `references/` 路径均相对于本 skill 根目录（即 `novel-editor/`）。

## 评测工作流

```
1. 接收文本 → 2. 确认评测参数 → 3. 通读分析 → 4. 逐维度评分 → 5. 生成评测报告 → 6. 输出修改建议
```

### 步骤 1：接收文本

支持三种输入方式：
- **文件路径**：读取指定的小说文件
- **直接粘贴**：用户直接提供文本
- **多文件**：大纲 + 正文组合输入

如文本超长无法一次性处理：可先评测大纲 + 前几章给出整体诊断与类型/权重建议，再分批评测其余章节并合并结论；或仅对用户指定的章节/局部做评测。优先保证单次评测范围可完整通读。

### 步骤 2：确认评测参数

与用户确认以下参数（未提供的项主动询问关键项，其余取默认值）：

- **评测模式**：
  - 全篇评测（默认）：对完整作品的系统评测
  - 章节评测：对单章或连续几章的详细评测
  - 局部评测：对特定段落/场景的聚焦评测
- **小说类型**：影响维度权重（读取 [references/genre-focus.md](references/genre-focus.md) 获取权重分配）
- **篇幅类型**：短篇 / 中篇 / 长篇（影响评测标准的侧重）
- **评测侧重**（可选）：用户特别关注的维度
- **作品阶段**：初稿 / 修订稿 / 终稿（影响评价严格度，初稿适当宽容）

### 步骤 3：通读分析

完整阅读文本，建立整体印象：
- 概括故事梗概（3-5句）
- 识别小说类型和风格倾向
- 标记初步发现的亮点和问题点
- 理解作者的创作意图

### 步骤 4：逐维度评分

从八个维度逐一评测打分（1-10分）。每个维度的详细评分标准参见 [references/rubric.md](references/rubric.md)。

**八大评测维度：**

| # | 维度 | 核心考察点 |
|---|------|-----------|
| 1 | 情节架构 | 因果逻辑、冲突设计、伏笔回收、节奏曲线、转折质量 |
| 2 | 角色塑造 | 立体度、动机合理性、弧线完整度、对话个性化、关系张力 |
| 3 | 文笔风格 | 语言精度、修辞质量、叙述声音、感官层次、描写效率、**修辞节制** |
| 4 | 世界观构建 | 内部一致性、融入方式、细节密度、原创程度 |
| 5 | 主题表达 | 深度、展示vs说教、与情节融合度、层次感 |
| 6 | 情感共鸣 | 代入感、感染力、情绪节奏、情感真实度 |
| 7 | 创新性 | 概念新颖度、叙事手法、类型突破、个人风格 |
| 8 | 可读性 | 开篇吸引力、翻页动力、信息密度、阅读流畅度 |

**评分规则：**
- 每个维度给出整数或0.5精度的分数
- 必须给出分数理由和具体文本引用
- 指出该维度的最大亮点和最大问题
- 根据小说类型应用对应权重计算综合分

### 步骤 5：生成评测报告

按以下模板输出评测报告。**篇幅**：若评测对象为文件，须先调用 MCP 工具 **count_document(正文路径)** 获取精确字数再填写「约 X 万字」，勿估算。

```
# 📖 小说评测报告

## 基本信息
- 作品名：
- 类型：
- 篇幅：约 X 万字（若评测对象为文件，须先调用 MCP 工具 **count_document(正文路径)** 获取精确字数后填写，勿估算）
- 评测模式：全篇评测 / 章节评测 / 局部评测
- 作品阶段：

## 故事梗概
[3-5句概括]

## 总体印象
[2-3段整体评价，先说优点再说问题，语气建设性]

## 维度评分

| 维度 | 分数 | 权重 | 加权分 |
|------|------|------|--------|

| 角色塑造 | X/10 | X% | X |
| 文笔风格 | X/10 | X% | X |
| 世界观构建 | X/10 | X% | X |
| 主题表达 | X/10 | X% | X |

## 修辞节制专项

| 检查项 | 统计 | 判定 |
|--------|------|------|
| "不是…是…"句式 | X次（X次/千字） | 正常 / 偏高 / 依赖 |
| "像…"明喻 | X次（X次/千字） | 正常 / 偏高 / 依赖 |
| 三项以上排比 | X处 | 正常 / 偏多 / 过度 |

### 三大亮点

2. [第二个优点]
3. [第三个优点]

### 三大问题
1. [最需要解决的问题，具体说明]
2. [第二个问题]
3. [第三个问题]
```

### 步骤 6：输出修改建议

修改建议按优先级分三级输出：

```
## 修改建议

### 🔴 必改项（影响作品根基）
[严重的结构性、逻辑性问题，不改会导致作品不成立]
每条建议格式：
- **问题**：具体描述
- **位置**：章节/段落定位
- **建议**：具体可执行的修改方案
- **示例**：如适用，给出修改示范

### 🟡 建议项（显著提升质量）
[修改后能明显提升阅读体验的问题]
格式同上

### 🟢 优化项（锦上添花）
[进一步打磨的空间]
格式同上

### 修改路线图
[建议的修改顺序和步骤，帮助作者有序地改稿]
```

**修改建议原则：**
- 每条建议必须具体、可执行，避免空泛的"加强描写""深化主题"
- 指出问题时必须引用原文
- 提供修改示范时保持作者的风格，不替换为评测者自己的风格
- 尊重作者意图，建议应在作者的创作方向上优化而非改弦易辙
- 区分"技术问题"和"风格选择"，后者仅提示而不强求修改

**报告与修改建议**：以 Markdown 形式输出；若用户指定保存路径（如工作区下某目录），则写入该路径，否则在对话中直接呈现。

## 评测模式详解

### 章节评测

用于评测单章或连续几章，在全篇评测的基础上增加：
- 本章在整体结构中的功能分析
- 章首衔接与章尾钩子评价
- 信息节奏分析（本章推进了多少新信息）
- 视角一致性检查

若用户仅提供单章而无大纲或前后文，需在报告中说明「整体结构信息有限」，本章评测侧重本章内部质量与可识别的衔接/钩子表现。

### 局部评测

用于聚焦评测特定段落或场景，侧重：
- 微观文笔分析（逐句点评）
- 场景构建效率
- 对话质量诊断
- 描写技法建议
- 可提供逐行批注式反馈：输出格式为「引用原文片段 + 段后/行侧批注」，便于作者对照修改

## 评分校准原则

- **锚定参照系**：心中以公认经典作品为 9-10 分，畅销佳作为 7-8 分进行锚定
- **避免集中效应**：不要所有维度都给 6-7 分，大胆拉开差距
- **类型公平**：不因个人文学偏好歧视通俗类型，网文有网文的评价标准
- **阶段适配**：初稿评测聚焦结构和逻辑，不苛求文笔细节；终稿评测全面严格
- **正向优先**：先认可亮点，再指出问题，保持建设性语气
